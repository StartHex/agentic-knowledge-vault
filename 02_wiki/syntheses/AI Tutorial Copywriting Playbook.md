## Purpose

This page turns high-performing AI videos, tutorials, and copywriting articles into reusable writing patterns. It should help the knowledge base behave like a copywriting assistant for AI tutorial accounts.

The goal is not to copy viral scripts. The goal is to distill:

- why the hook works
- how the story is arranged
- where proof appears
- how technical content is simplified
- what makes the viewer want the next step

## Distillation Unit

Every collected script or article should be reduced into these fields:

- **Topic**: what the content is about.
- **Audience pain**: what uncertainty, fear, desire, or job-to-be-done it targets.
- **Hook type**: curiosity, loss, contrast, authority test, shortcut, mistake, controversy, result-first.
- **Promise**: the concrete benefit the viewer expects.
- **Proof**: screenshot, live demo, metric, source, before/after, failure recovery.
- **Structure**: opening, setup, conflict, steps, payoff, CTA.
- **Reusable sentence patterns**: abstracted templates, not copied wording.
- **Risk**: exaggeration, unsupported claim, policy/compliance issue, outdated steps.
- **Adaptation idea**: how to rewrite it for the user's AI handbook/account.

## Hook Patterns

### 1. Result First

Best for tool demos and automation tutorials.

Template:

```text
我用 X 做完了 Y，原来要 Z 分钟，现在只要 N 步。
```

Examples to adapt:

- "我用 Codex 自动改完一个项目"
- "我把 AI 新闻自动同步进 Obsidian 图谱"
- "我让 n8n 每天生成选题清单"

### 2. Hidden Cost

Best for debunking hype.

Template:

```text
大家都在说 X 强，但真正卡住普通人的不是 X，而是 Y。
```

Use for:

- API key / quota / network / path issues
- tools that look easy but have setup traps
- "国产 Codex" comparison topics

### 3. Mistake Avoidance

Best for technical tutorials and setup content.

Template:

```text
如果你照着教程做却跑不起来，大概率是漏了这一步。
```

Good for:

- environment variables
- port conflicts
- launchd/systemd timers
- Obsidian vault path and graph visibility

### 4. Authority Test

Best when a large creator or official release is trending.

Template:

```text
大号都在讲 X，我只测一个问题：普通人能不能复现。
```

This keeps the account skeptical and practical.

### 5. Workflow Reframe

Best for building a durable account identity.

Template:

```text
这不是又一个 AI 工具，它改变的是这条工作流。
```

Use when turning news into a system:

- AI news -> knowledge base
- tutorial -> reusable manual
- transcript -> script draft
- daily collection -> creator action plan

## Script Structures

### Short Video: 60-90 Seconds

```text
0-3s: show result or pain
3-8s: name the hidden problem
8-20s: show the tool/workflow
20-50s: three concrete steps
50-70s: one pitfall and fix
70-90s: who should use it and next topic
```

### Tutorial Video: 5-10 Minutes

```text
1. Result demo
2. Why this matters now
3. Tool choice and prerequisites
4. Step-by-step build
5. Real error or limitation
6. Final verification
7. How to extend it
```

### Article / Thread

```text
1. One-sentence conclusion
2. Context: what changed
3. Why it matters to the target user
4. Reproducible workflow
5. Mistakes and tradeoffs
6. Checklist
7. Recommended next action
```

## Sentence Pattern Library

Use these as rewrite templates:

- "我不是来复述新闻的，我直接拿一个真实任务测。"
- "这个工具真正有价值的地方，不是能聊天，而是能接管这一步。"
- "如果你每天都在看 AI 新闻，但没有沉淀成手册，其实第二天就忘了。"
- "这一步看起来小，但决定了后面能不能自动化。"
- "不要先问哪个模型最强，先问这个任务能不能闭环。"
- "我把它拆成三层：采集、判断、沉淀。"
- "普通人真正需要的不是更多工具，而是一套可复用流程。"
- "这类教程最容易误导人的地方，是只展示成功画面，不展示失败条件。"

## Copy Review Checklist

Before publishing:

- Does the first sentence create a concrete question?
- Is the promise visible and testable?
- Is the audience clearly identified?
- Is there one real proof point before heavy explanation?
- Are there fewer than three main ideas?
- Is the technical part converted into steps or labels?
- Is the ending connected to an ongoing series?
- Does the script avoid copying any source wording?

## Source Types To Collect

High value:

- viral AI tutorial videos with transcripts
- high-read AI tool articles
- copywriting breakdown articles
- creator postmortems and performance reviews
- official case studies with concrete workflows

Lower value:

- pure motivational content
- generic prompt lists without examples
- hype-only news with no reproducible workflow
- content that depends on private access or unverifiable claims

## Knowledge Base Integration

When a strong source is collected:

1. Create a source note under `02_wiki/source-notes/`.
2. Extract the distillation unit fields.
3. Link useful mechanisms to this page, [[AI Tutorial Editing Playbook]], and [[AI Tutorial Psychology Playbook]].
4. Add one adapted script idea to the creator idea backlog.
5. If the pattern repeats across sources, promote it into a stable template here.
