# github-devpor

`github-devpor` 是一个基于 Electron + React 的 GitHub 项目助手，聚焦在国内网络环境下的“搜索 -> 阅读 -> 下载 -> 安装”闭环体验。  
项目当前以客户端能力为核心，强调可用性、稳定性与低学习成本。

## 项目亮点

- 一体化工作流：在同一窗口完成项目搜索、README 阅读、下载与安装操作
- 国内网络友好：内置代理候选与失败重试机制，降低 GitHub 资源访问门槛
- 双模式安装：支持普通模式与懒人模式，兼顾灵活控制和效率
- AI 辅助安装：懒人模式下可生成最小安装步骤并逐条执行
- 安全执行机制：命令预览、黑名单拦截、超时控制，减少误操作风险
- 深色卡片式 UI：信息分区清晰，适合长时间使用和状态观察

## 主要功能

### 1) GitHub 项目检索

- 支持关键词搜索与项目 URL 直达
- 搜索结果结构化展示，便于快速筛选目标仓库

### 2) README 内嵌阅读

- 在应用内直接查看仓库 README
- 支持链接跳转与阅读跟随，减少上下文切换

### 3) 下载与代理加速

- 支持源码包与 Release 资源下载
- 代理异常时自动切换并重试，提高下载成功率
- 提供下载进度与任务状态反馈

### 4) 安装执行

- 普通模式：下载后由用户按需手动安装
- 懒人模式：自动生成并执行最小安装步骤
- 支持执行日志查看，便于排查失败原因

### 5) 配置与本地存储

- 支持模型来源、代理策略、安装模式等参数配置
- 关键配置本地持久化，重启后可延续使用

## 设计说明

### 设计目标

- 降低“找到并安装开源项目”的操作门槛
- 在复杂网络环境中保持流程稳定
- 让用户在自动化与手动控制之间自由切换

### 界面结构

- 顶部信息区：当前模式与关键状态
- 参数配置区：模型、代理、安装策略等设置
- 主工作区：左侧列表（项目/任务），右侧详情（README/日志）
- 底部日志区：统一展示下载、安装和运行反馈

### 交互原则

- 优先可见：关键状态和错误原因尽量直接展示
- 最小打断：失败自动重试，必要时再请求用户决策
- 渐进增强：先满足基础流程，再叠加 AI 与自动化能力

## 技术栈

- 桌面端：`Electron`、`TypeScript`
- 前端：`React 18`、`Vite`、`Zustand`
- 进程通信：`ipcMain` / `ipcRenderer`
- 本地存储：`electron-store`
- 打包发布：`electron-builder`

## 快速开始

### 安装依赖

```bash
npm install
```

### 启动开发

```bash
npm run dev
```

### 构建与打包

```bash
npm run build
npm run dist
```

## 配置项（客户端）

- `mode`: `normal` / `lazy`
- `aiProvider`: `builtin` / `custom`
- `aiBaseUrl`、`aiApiKey`、`aiModel`（使用自定义模型时填写）
- `proxyMode`: `auto` / `custom`
- `customProxy`（可选）

## 安全与稳定性

- 命令执行前可查看具体步骤
- 高危命令黑名单拦截
- 执行超时与输出体积限制
- 下载失败自动切换代理重试
